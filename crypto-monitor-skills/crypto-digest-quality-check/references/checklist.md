# QA Checklist — детальный чек-лист

## 1. Copyright (защита от копирайта)

- [ ] Ни одна последовательность из 15+ слов в summary не совпадает дословно с body источника.
- [ ] Ни в одной публикации нет более одной короткой цитаты (≤15 слов) из одного источника.
- [ ] Все короткие цитаты в кавычках имеют чёткую атрибуцию («по словам X», «по формулировке X»).
- [ ] Структура summary не повторяет структуру body источника абзац-в-абзац.
- [ ] Тексты регуляторов (SEC, MAS, AFSA, НБРК) пересказаны своими словами, не процитированы целиком.

## 2. Factual sanity (фактическая корректность)

- [ ] Все даты валидны (нет 30 февраля, 32 марта и т.п.).
- [ ] Все денежные суммы выражены в корректном формате ($1 млрд, 100 млн тенге).
- [ ] Названия регуляторов и компаний написаны корректно (SEC, не S.E.C.; AFSA, не AFTA).
- [ ] Имена ключевых лиц (если упомянуты) — без явных опечаток.
- [ ] Цифры в summary совпадают с цифрами в исходных данных (с учётом округления до 10%).
- [ ] Юрисдикции согласованы (например, событие в США → country=US, а не SG).

## 3. Tonal (тональность)

- [ ] Нет сенсационных эпитетов: «шокирующий», «эпохальный», «беспрецедентный», «прорывной», «провальный».
- [ ] Нет восклицательных знаков.
- [ ] Нет вопросительных знаков (кроме случаев, когда вопрос — часть прямой речи).
- [ ] Нет эмоциональных оценок («впечатляющий рост», «катастрофическое падение»).
- [ ] Нет «своих прогнозов» сводки («ожидается, что…», «вероятно, последует…») — если этого не говорят в источнике.
- [ ] Нет рекламно-окрашенных формулировок.
- [ ] Нет жаргона («крипта», «битки», «памп», «дамп»).

## 4. Structural (структурная целостность)

- [ ] В шапке указаны: дата сводки, период покрытия, общее число публикаций.
- [ ] Подвал содержит дисклеймер, ссылку на архив, ссылку на управление подпиской.
- [ ] Нет дубликатов публикаций (по id или по URL).
- [ ] Нет пустых разделов с заголовком, но без публикаций.
- [ ] Все обязательные поля заполнены у каждой публикации (title_ru, summary, source_name, source_url, priority).

## 5. Prioritization (правильность приоритизации)

- [ ] При наличии geo_priority=1 публикаций раздел «Регулирование РК» — первый раздел.
- [ ] Внутри каждого раздела публикации отсортированы по score убыванию.
- [ ] Все critical-публикации находятся в первой половине своего раздела.
- [ ] Если в сводке есть critical, она помечена соответствующим маркером.

## 6. Format (форматирование)

### HTML
- [ ] Все открытые теги закрыты.
- [ ] Все цвета — в HEX-формате.
- [ ] Все стили — inline (нет внешних таблиц стилей).
- [ ] Ширина контейнера фиксированная (640px) — для Outlook.
- [ ] Все URL — абсолютные, начинаются с https://.

### Telegram MarkdownV2
- [ ] Все спецсимволы экранированы обратным слэшем: `_ * [ ] ( ) ~ \` > # + - = | { } . !`.
- [ ] Markdown-разметка корректна (открытие/закрытие звёздочек, ссылок).
- [ ] Длина каждого сегмента ≤ 4000 символов.
- [ ] Промежуточные сегменты содержат пометку «продолжение далее» или «(продолжение)».
- [ ] Не разорваны блоки публикаций между сегментами.

### Plain text
- [ ] Нет HTML-тегов.
- [ ] Все URL читаемы.
- [ ] Структура читается без форматирования.

## 7. Length (длина)

- [ ] Каждая summary — 60–120 слов.
- [ ] Каждый title_ru — 30–100 символов.
- [ ] Общая длина plain_text — не более 100 КБ.
- [ ] Раздел не более `max_items_per_section` (по умолчанию 5) публикаций.

## 8. Links (ссылки)

- [ ] У каждой публикации указан source_url.
- [ ] Все URL открываются (опционально — проверка через HEAD-запрос, см. note ниже).
- [ ] URL не ведут на агрегаторы (ya.news, ru.investing.com), а на оригинал.
- [ ] URL не содержат tracking-параметров (utm_source и т.п.).

*Примечание.* Проверка фактической доступности URL — медленная операция; рекомендуется делать её отдельным джобом, не в рамках skill.

## 9. Источники (sources)

- [ ] Все source_name — нормализованные домены или официальные названия.
- [ ] Источников с очевидно низким доверием (анонимные Telegram-каналы) — не более 20% в сводке.

## 10. Финальная проверка

- [ ] Сводка читается за 3–5 минут.
- [ ] Любой получатель из целевой аудитории поймёт каждое сообщение без обращения к оригиналу.
- [ ] Сводка не содержит технического жаргона без пояснений.
