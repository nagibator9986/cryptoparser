# crypto-news-deduplicator

## Назначение

Skill принимает массив публикаций за окно ±48 часов и кластеризует их: какие описывают одно и то же событие. Для каждого кластера выдаёт каноническую публикацию (самый достоверный источник) и список остальных в кластере.

## Когда использовать

- На этапе обработки большого пула публикаций перед сборкой сводки.
- Когда регулятор выпустил релиз и десятки СМИ его перепечатали.
- Когда событие развивается (исходный анонс + комментарии + реакция рынка).

## Контракт

### Вход

```json
{
  "articles": [
    {
      "id": "art_001",
      "title": "...",
      "body": "...",
      "source_name": "...",
      "source_url": "...",
      "published_at": "ISO 8601",
      "language": "ru",
      "topics": ["regulation", "cbdc"],
      "country": "KZ"
    }
  ]
}
```

### Выход

```json
{
  "clusters": [
    {
      "cluster_id": "c1",
      "canonical_id": "art_001",
      "member_ids": ["art_001", "art_007", "art_012"],
      "event_summary": "Запуск второй фазы пилота цифрового тенге",
      "rationale": "Все статьи описывают объявление НБРК от 12 марта о подключении 4 банков к пилоту."
    }
  ],
  "singletons": ["art_002", "art_003"]
}
```

## Алгоритм

1. **Сравни пары статей** по трём якорям: главный субъект, тип действия, ключевая дата или цифра.
2. **Считай дубликатами**, если совпадают все три якоря или два из трёх плюс высокое сходство заголовков.
3. **Не объединяй** статьи только по совпадению тематики (одна тема ≠ одно событие).
4. **Выбери канонический источник** по иерархии: регулятор > национальное СМИ > крипто-СМИ > соцсети. При прочих равных — более ранняя по `published_at`.
5. **Сформулируй event_summary** в одном предложении на русском.

## Правила

- **Якорь 1: субъект.** Главное действующее лицо/организация (SEC, НБРК, конкретная биржа).
- **Якорь 2: тип действия.** Что произошло (запуск пилота, штраф, лицензия).
- **Якорь 3: дата или цифра.** Точная дата события или ключевая цифра (сумма штрафа, объём пилота).
- **Не объединяй** анонс события и сам факт события — это два разных события (вчера: «планируется», сегодня: «состоялось»).
- **Не объединяй** материал и его комментарий — комментарий аналитика отдельная новость.
- **Объединяй** перепечатки с цитированием первоисточника.

## Иерархия источников для канонизации

1. **Официальные регуляторы и центральные банки** (sec.gov, nationalbank.kz, mas.gov.sg) — максимальный приоритет.
2. **Официальные сайты компаний-субъектов** (binance.com/blog, для новости о Binance).
3. **Авторитетные деловые СМИ** (Bloomberg, Reuters, Forbes Kazakhstan, Курсив).
4. **Специализированные крипто-СМИ** (CoinDesk, The Block, Forklog).
5. **Социальные сети и Telegram-каналы** — минимальный приоритет; не выбираются как канонические, если есть альтернатива.

## Примеры

### Пример 1: явный кластер

Вход:
- art_001: «НБРК объявил о запуске 2-й фазы пилота цифрового тенге» (nationalbank.kz, 12 марта)
- art_002: «В Казахстане стартовала вторая фаза цифрового тенге» (kapital.kz, 12 марта)
- art_003: «Halyk, Kaspi, Forte и Jusan присоединились к пилоту цифрового тенге» (forklog.com, 13 марта)
- art_004: «Биржа Binance отменила листинг токена X» (binance.com, 12 марта)

Выход:
```json
{
  "clusters": [
    {
      "cluster_id": "c1",
      "canonical_id": "art_001",
      "member_ids": ["art_001", "art_002", "art_003"],
      "event_summary": "Запуск второй фазы пилота цифрового тенге с участием четырёх банков",
      "rationale": "Все три статьи описывают одно объявление НБРК от 12 марта; субъект — НБРК, действие — запуск второй фазы, дата — 12 марта."
    }
  ],
  "singletons": ["art_004"]
}
```

### Пример 2: тематическое сходство, но не один кластер

Вход:
- art_010: «SEC обвинила биржу A в работе без лицензии» (sec.gov, 5 марта)
- art_011: «MAS отозвало лицензию биржи B» (mas.gov.sg, 5 марта)

Выход:
```json
{
  "clusters": [],
  "singletons": ["art_010", "art_011"]
}
```

Объяснение: разные субъекты (разные биржи, разные регуляторы) — два разных события, несмотря на тематическое сходство.

## Антипримеры

❌ Объединение анонса («НБРК планирует запустить пилот») и факта («НБРК запустил пилот») — это две разные новости.

❌ Объединение по теме («все новости про CBDC за день в один кластер»).

## Граничные случаи

- **Анонимные источники vs официальные** → если новость подтверждена официально, канонизируй официальный источник; анонимный source оставь в member_ids с пометкой в rationale.
- **Развивающаяся история (live updates)** → каждый существенный этап — отдельное событие.
- **Один и тот же URL встретился дважды** → это технический дубль, считай одной статьёй.
